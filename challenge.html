Challenge

Proposed by Marian Veteanu

I hope you liked the previous week's recreational coding challenge! I hope you found it fun and refreshing. This week we are proposing a similar challenge. If you have time please try to implement it, so we can gave a real game jam!

In this coding challenge you continue your journey of game dev wannabe, and plan to produce and release a clone of the classical game "Snake".

Of course, as a timid player in this space, you’ll probably want to work on this challenge on a Friday evening, and then share with your close colleagues first, for feedback.

As always, you plan to write the game using only HTML / JavaScript, on an 800x600 Canvas element.


The game mechanics are pretty simple:

  • You have to direct a small snake of 1 unit on the a grid-like system on top of a Canvas (everything on this screen will be represented on this grid).
  • The snake slithers automatically at a certain speed, without pressing any key. Pressing arrow keys makes the snake change direction appropriately.
  • A random piece of food appears on the screen.
  • Each time the snakes eats the food, it grows with 1 unit.
  • You need to avoid crashing into walls, as well as into itself (this becomes more and more challenging, as the snake grows).

For you as a developer, this game poses 3 challenges:

  1. How to implement the snake? (Hint: arrays)
  2. How to grow the snake after eating food? (Hint: the snake grows at the head, not at the tail)
  3. How to work with the canvas and game loop? (Hint: see last week's challenge)

Good luck, and enjoy!!


Note: After you finish the coding, show the game to your kids or family… See what they think about it! Seek their opinion on how to improve the game mechanics or graphics!
